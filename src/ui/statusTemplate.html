Vaultage

VAULTAGE is a white-label alternative to Dropbox/Google Drive, providing secure JWT-based auth in conjunction with S3 handshakes to facilitate the transit of bytes from system to Supabase, whilst never handling a byte on the server.

01. Operation Modes

The system's behavior is controlled via the VLT_ALLOW_REAL_UPLOADS environment variable.

Restricted Mode (Default)

Only the example.txt handshake file is permitted. All other uploads are rejected at the controller level.

Production Mode

Full binary support is enabled. Users can upload PDFs and standard assets up to a 5MB limit.

02. The Test Upload

To verify the storage pipeline without consuming quota, the system uses a "Canary" file protocol:

  • Filename: Must be exactly example.txt
  • File Size: Must be 0 bytes (empty)

03. Cleanup & Deletion

Clicking the ✕ button triggers a dual-wipe protocol. The system removes the physical binary from Supabase storage and simultaneously drops the metadata record from the PostgreSQL database.

Future Scope

This Proof of Concept currently utilizes a static directory structure. Production iterations will support dynamic folder creation, recursive file moving, and custom metadata tagging.

Security Constraint

POC Restriction: Only an empty example.txt will bypass current RLS validation.
